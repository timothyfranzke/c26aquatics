Background treatments to add

Two devices, applied only where a light section meets a dark one. On the homepage that's two seams: into Coaches (after the white Tracks) and into the final CTA (after the white FAQ). Rule of thumb: one divider + one texture per seam, never stacked.

A · Hatch-band divider

A 44px band reusing the hero's 45° stripe, placed immediately before each dark section. Cleanest as a tiny presentational component dropped into index.astro.

/* new: components/sections/SeamHatch.astro */
<div class="seam-hatch" aria-hidden="true"></div>
<style>
  .seam-hatch {
    height: 44px;
    background: repeating-linear-gradient(
      45deg, var(--hatch-a) 0 11px, var(--hatch-b) 11px 22px);
  }
</style>

/* index.astro — place before each dark section */
<SeamHatch /> <CoachesGrid ... />
<SeamHatch /> <CTABand ... />
B · Kraken watermark

Oversized kraken-white.png bleeding off-edge at very low opacity — the same trick the hero already uses. Add to the dark sections only (Coaches bottom-left ~.05, Final CTA right ~.06).

/* on the dark section root */
.coaches--dark { position: relative; overflow: hidden; }

.coaches--dark::after {            /* or an <Image> el */
  content: "";
  position: absolute;
  left: -80px; bottom: -80px;
  width: 340px; height: 340px;
  background: url('/kraken-white.png') center/contain no-repeat;
  opacity: .05;
  pointer-events: none;
}
/* keep section content above it: */
.coaches--dark .container { position: relative; z-index: 1; }

Note: the stripe navies #1a2150 / #26306c aren't in tokens/colors.css yet — add --hatch-a / --hatch-b there so the seam is themeable.
